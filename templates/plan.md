## Planning Rules

- Each task has one outcome.
- Slice vertically so every task leaves something working.
- Acceptance criteria describe outcomes, not implementation steps.
- Use three acceptance criteria or fewer per task.
- Every task has a concrete verification command or manual check.
- Every completed task should be reviewable as its own commit.
- If a task is hard to review, split it.
